## 🔗 LINKING
> 수많은 정보와의 싸움에서 이겨라, **LINKING**의 프론트엔드 레포지토리

**링크를 효율적으로 저장하고 관리할 수 있게 해주는 아카이빙 시스템**입니다.

### 🦴 브랜치 컨벤션
> 브랜치 이름을 지정할 때의 규칙입니다.

- 영어로 통일, kebab-case로 작성
- 작성 형식 : 브랜치 유형/#이슈번호-브랜치명
- ex :
	- feature/login
	- feature/#3-navbar
	- docs/#32-README.md

### 📝 커밋 컨벤션
> 커밋 시 어떤 내용을 추가/수정했는지 나타내는 규칙입니다.
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 스타일 변경
- `design`: UI 디자인 변경
- `test`: 테스트 코드
- `refactor`: 리팩토링
- `ci`: CI 설정 파일 수정
- `perf`: 성능 개선
- `chore`: 빌드, 패키지 설정 등 기능과 직접 관련 없는 작업
- `rename`: 파일 혹은 폴더명을 수정만 한 경우
- `remove`: 파일을 삭제만 한 경우

### 📐 코드 컨벤션
> 코드 네이밍 룰로, 이름을 지을 때의 규칙입니다.

- PascalCase : 컴포넌트 함수, 클래스, 타입, 인터페이스, 파일명
- camelCase : 변수, 일반함수
- UPPER_CASE : 상수
